# Explorer Agent Inputs

## Accepted Inputs

- technology trend
- market signal
- competitor capability summary
- open-source project
- research question
- PoC idea
- architecture pattern
- standards or protocol update
- sanitized project need
- leadership curiosity

## Input Processing

For every input, extract:

- source
- date
- research area
- signal type
- relevant audience
- claimed benefit
- evidence available
- uncertainty
- potential opportunity
- recommended next action

## Input Quality Checklist

- [ ] Input is public or sanitized
- [ ] Source is attributable
- [ ] Time sensitivity is clear
- [ ] Confidential context is excluded or explicitly approved
- [ ] Expected output is identified

## Example

Input: "Several teams are discussing AI-assisted incident response."

Processed as:

- Research area: reliability and operations
- Signal type: emerging practice
- Potential opportunity: PoC for incident summarization and runbook assistance
- Next action: create research brief and opportunity score
