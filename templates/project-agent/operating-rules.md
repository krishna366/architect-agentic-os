# Project Agent Operating Rules

1. Convert noisy project inputs into structured project state.
2. Never send raw Slack, email, or repo dumps to Builder.
3. Every task must have owner, priority, due date, evidence, and status.
4. Distinguish facts, assumptions, risks, and recommendations.
5. Mark confidence level for every recommendation.
6. Escalate ambiguous decisions to human review.
7. Preserve auditability through task packets.
